# GitHub Publishing Checklist

## Files To Upload

Upload the complete `dsmb-training-prototype` folder contents:

- `index.html`
- `app.js`
- `styles.css`
- `assets/`
- `data/`
- `README.md`
- `OPEN_ME_FIRST.txt`
- `.gitignore`

Keep the folder structure unchanged. Do not upload only `index.html`.

## Create The Repository

1. Create a new GitHub repository.
2. Choose the visibility approved by your organization.
3. Upload all files and folders listed above to the repository root.
4. Commit the uploaded files to the `main` branch.

## Enable GitHub Pages

1. Open the repository's **Settings**.
2. Select **Pages**.
3. Under **Build and deployment**, choose **Deploy from a branch**.
4. Select the `main` branch.
5. Select the `/ (root)` folder.
6. Select **Save**.

GitHub will provide the published URL after deployment completes.

## Verify The Published Training

Confirm that:

- The training homepage loads.
- Images and fonts appear.
- Micro-trainings start and advance.
- Text fields accept typing.
- Dropdown choices are visible and selectable.
- Final actions close each training.

## Updating The Published Version

Upload and commit the changed files to the same repository and branch.
GitHub Pages will publish the new version automatically.

Do not publish synthetic training content publicly unless the repository
visibility and hosting approach have been approved by your organization.
